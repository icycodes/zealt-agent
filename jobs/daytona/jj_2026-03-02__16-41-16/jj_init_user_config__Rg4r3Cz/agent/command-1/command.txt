## Background

`jj` (Jujutsu) is a next-generation version control system that is compatible with Git repositories. Unlike Git, every working-copy state is automatically tracked as a commit, and user identity is configured via `jj config set`.

Configuration in `jj` lives in several layers:
- **User-level** (`--user`): stored in `~/.config/jj/config.toml` (on Linux/macOS). Applies to all repositories for the current user.
- **Repo-level** (`--repo`): stored inside the repository'"'"'s `.jj/` directory. Overrides user-level config for that repo.

## Tips for New jj Users

- `jj` automatically snapshots your working copy; you don'"'"'t need to `git add` files.
- The working copy is always represented as a commit (`@` in `jj log`).
- The `jj config set` command writes directly to your config file — no editor needed.
- After setting your identity, the author/committer fields in `jj log` should reflect your configured name and email.
- You can confirm settings with `jj config list user` to see all user-scoped config keys.
